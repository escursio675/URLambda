#Elicitation

## Requirements Elicitation

In our Agile methodology, requirement elicitation is a continuous and collaborative process rather than a single, upfront phase. Needs and expectations are gathered iteratively, discussed during Sprint Planning, and captured in our Product Backlog as Epics and User Stories

### Core Epics and User Stories (Initial Backlog)

Based on initial elicitation, we have translated the primary user needs into the following Agile User Stories

#### Epic 1: Core Link Generation & Redirection

*Story 1.1:* As a casual user, I want to paste a long URL and click a single button so that I can instantly receive a shortened, easily shareable link.

*Story 1.2:* As an end-user clicking a shared link, I want the system to redirect me to the original destination with near-zero latency so that my browsing experience is seamless.

*Story 1.3:* As the system administrator, I want the Express backend to validate all submitted URLs so that malicious or broken links are not saved to our MongoDB database.

#### Epic 2: UI

*Story 2.1:* As a user, I want a "Copy to Clipboard" button next to my generated links so that I can quickly paste them into my social media or emails.
